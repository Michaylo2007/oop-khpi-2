# Лабораторна робота №1 — Реєстрація пацієнтів приватної клініки

## Опис предметної галузі

Приватна клініка надає медичні послуги пацієнтам через лікарів різних спеціалізацій.
Для нормальної роботи клініки необхідно вести облік пацієнтів, лікарів та записів на прийом.

Кожен **пацієнт** має унікальний ідентифікатор, повне ім'я, рік народження та прапор
наявності активного запису. Пацієнт може бути призначений до конкретного лікаря.

Кожен **лікар** має унікальний ідентифікатор, ім'я, спеціалізацію та статус доступності.
Лікар може приймати пацієнтів лише якщо він доступний.

**Запис на прийом** (Appointment) пов'язує конкретного пацієнта з конкретним лікарем
на певну дату. Запис має статус: `SCHEDULED`, `COMPLETED` або `CANCELLED`.

## Компіляція і запуск

```bash
# Компіляція
javac -d out src/main/java/ua/khpi/oop/lab01/*.java

# Запуск
java -cp out ua.khpi.oop.lab01.Main
```

## Вибір полів для equals() і hashCode()

- `Patient` — рівність за `patientId`: ідентифікатор унікально визначає пацієнта в системі.
- `Doctor` — рівність за `doctorId`: аналогічно, унікальний ідентифікатор лікаря.
- `Appointment` — рівність за `appointmentId`: кожен запис унікальний.

Ім'я чи рік народження не використовуються для рівності, оскільки можуть збігатися у різних людей.

## Можливий розвиток моделі

У наступних лабораторних роботах модель можна розширити:
- Додати клас `Clinic` як контейнер для списків лікарів і пацієнтів (колекції Java).
- Додати ієрархію лікарів через успадкування (`Specialist extends Doctor`).
- Реалізувати пошук вільних лікарів за спеціалізацією.
